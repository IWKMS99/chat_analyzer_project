# Руководство пользователя (Русский)

## Что делает инструмент

Chat Analyzer читает JSON-экспорт из Telegram Desktop и строит интерактивный дашборд с:

- метриками сообщений и активности,
- временной и пользовательской статистикой,
- графиками и таблицами для анализа.

## Быстрый запуск (Docker, готовые образы)

```bash
docker compose -f docker-compose.prod.yml up -d
```

Открыть в браузере:

- Web UI: `http://localhost:8080`
- API docs: `http://localhost:8080/docs`

## Как получить `result.json` из Telegram

1. Откройте **Telegram Desktop**.
2. Перейдите в **Settings -> Advanced -> Export Telegram data**.
3. Выберите формат экспорта **JSON**.
4. Отметьте нужные чаты/сообщения.
5. Для первых запусков лучше отключить медиа, чтобы уменьшить размер файла.
6. Запустите экспорт и дождитесь завершения.
7. Откройте папку экспорта и найдите JSON-файл (часто `result.json`).

## Загрузка и анализ

1. Откройте web-интерфейс Chat Analyzer.
2. Перейдите на экран анализов и выберите загрузку файла.
3. Загрузите Telegram JSON (`result.json` или аналогичный файл).
4. Дождитесь статуса `done`.
5. Откройте дашборд.

## Troubleshooting

### Ошибка загрузки (HTTP 413)

- Увеличьте `MAX_UPLOAD_BYTES` в `.env`.
- Перезапустите стек после изменения переменных.

### Анализ выполняется медленно

- Увеличьте `TASK_WORKERS` (текущая валидация ограничивает до 4).
- При необходимости настройте `CHAT_ANALYZER_NLP_WORKERS` под доступные CPU.

### Предупреждения fallback-парсинга или высокая нагрузка памяти

- Настройте `CHAT_ANALYZER_FALLBACK_JSON_MAX_BYTES`.
- Для диагностики начните с меньших чатов.

### UI не может обратиться к API

- Проверьте `VITE_API_BASE_URL` и путь reverse-proxy.
- Проверьте health endpoint: `http://localhost:8080/api/healthz`.

### Проблемы с локалью/отображением текста

- Экспортируйте файл напрямую из Telegram Desktop без перекодирования.
- Не редактируйте экспорт в редакторах, которые меняют кодировку.
